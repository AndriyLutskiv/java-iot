# Вступ

Мова програмування Java займає одне з провідних місць серед технологій розробки програмного забезпечення у світі. Завдяки принципу «напиши один раз — запускай будь-де» (*Write Once, Run Anywhere*), реалізованому через платформу JVM (Java Virtual Machine), Java широко застосовується у корпоративних застосунках, системах Інтернету речей (IoT), мобільних платформах (Android), хмарних сервісах та наукових обчисленнях. Станом на 2025 рік JDK 25 містить широкий спектр сучасних можливостей: записи (*records*), запечатані класи (*sealed classes*), розпізнавання зразків (*pattern matching*), текстові блоки (*text blocks*), віртуальні потоки (*virtual threads*) та структуровану паралельність (*structured concurrency*).

Метою дисципліни **«Програмування мовою Java»** є формування у студентів практичних навичок розробки програмного забезпечення з використанням мови Java, включаючи об'єктно-орієнтоване проєктування, роботу з колекціями та файловою системою, побудову мережевих застосунків, багатопотокове програмування та взаємодію з базами даних через JDBC.

Цей комплект методичних вказівок охоплює десять лабораторних робіт загальним обсягом 36 академічних годин та розрахований на студентів 2-го курсу спеціальності «Комп'ютерна інженерія» (F7). Передбачається, що студенти мають базові знання з алгоритмізації, можуть складати прості алгоритми та мають уявлення про структуру програм (мова C або подібні). Знання мови Java на початку дисципліни не обов'язкове.

## Структура методичних вказівок

Методичні вказівки складаються з таких розділів:

1. **Перелік використаних скорочень** — довідковий розділ із поясненням усіх абревіатур.
2. **Вступ** (цей розділ) — мета дисципліни, структура та вимоги до оформлення.
3. **Зміст** — навігаційний перелік усіх розділів.
4. **Лабораторні роботи №1–10** — основна частина, кожна містить: тему, мету, теоретичні відомості, хід виконання, індивідуальні завдання (15 варіантів), контрольні запитання та перелік літератури.

Практичні приклади у методичних вказівках пов'язані з відкритим навчальним репозиторієм `java-iot` (GitHub: `https://github.com/AndriyLutskiv/java-iot`), що містить реальні Java-проєкти для одноплатного комп'ютера CrowPi/Raspberry Pi. Студентам рекомендується переглядати вихідний код цих проєктів як зразок організації коду перед виконанням власних завдань.

## Вимоги до оформлення звіту

Звіт з кожної лабораторної роботи оформляється у вигляді текстового документа (формат ODT або PDF) і повинен містити:

1. **Титульний аркуш** — назва університету, кафедра, дисципліна, номер та тема лабораторної роботи, прізвище та ім'я студента, номер групи, рік.
2. **Мета роботи** — одне речення відповідно до методичних вказівок.
3. **Теоретична частина** — стислий конспект (3–5 абзаців) опрацьованого матеріалу; не переписування тексту методичних вказівок, а власне розуміння.
4. **Хід виконання** — опис кожного кроку з поясненням прийнятих рішень.
5. **Вихідний код програми** — повний текст написаних класів у форматі моноширинного шрифту (Courier New, 10 pt).
6. **Результати виконання** — знімки екрану або текстовий вивід програми, що підтверджують коректність роботи.
7. **Відповіді на контрольні запитання** — розгорнуті відповіді (по 3–5 речень) на обрані 5 запитань із переліку.
8. **Висновки** — 3–5 речень щодо набутих навичок та результатів роботи.

Код оформляється відповідно до стандарту Google Java Style Guide: відступ 2 пробіли, відкриваюча дужка на тому ж рядку, коментарі документації (`/** */`) для публічних класів та методів.
