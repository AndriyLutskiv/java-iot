# Лабораторна робота №1
## Ознайомлення з компонентами Java SDK. Перша програма: компіляція і виконання

---

### Тема

Ознайомлення з компонентами Java Development Kit (JDK), засобами компіляції та виконання Java-програм у командному рядку.

---

### Мета

Набути практичних навичок встановлення та налаштування JDK 25, написання, компіляції та запуску першої програми мовою Java шляхом безпосередньої роботи з інструментами командного рядка `javac` та `java`.

---

### Теоретичні відомості

#### Архітектура платформи Java

Платформа Java складається з трьох ключових компонентів:

- **JDK (Java Development Kit)** — повний набір засобів для розробки: включає компілятор `javac`, інструмент документування `javadoc`, налагоджувач `jdb`, засоби профілювання та JRE.
- **JRE (Java Runtime Environment)** — середовище виконання: містить JVM та стандартну бібліотеку класів. Починаючи з JDK 11, окремий дистрибутив JRE більше не поставляється — JDK використовується безпосередньо.
- **JVM (Java Virtual Machine)** — віртуальна машина, що виконує байт-код. Саме JVM забезпечує незалежність від платформи.

#### Модель компіляції та виконання

```
Вихідний код (.java)
        ↓  javac (компілятор)
Байт-код (.class)
        ↓  java (JVM)
Виконання програми
```

Компілятор `javac` перетворює текст програми (`.java`) на платформо-незалежний байт-код (`.class`). Інтерпретатор JVM виконує байт-код, перетворюючи його на машинні інструкції цільової системи (JIT-компіляція).

#### Структура програми Java

```java
// Ім'я файлу: HelloWorld.java
package ua.edu.example; // необов'язковий оголошувач пакету

// Клас — основна структурна одиниця Java
public class HelloWorld {

    // Точка входу: метод main
    public static void main(String[] args) {
        // Виведення рядка на стандартний вихід
        System.out.println("Привіт, Java 25!");
    }
}
```

**Ключові правила:**
- Ім'я публічного класу повинно збігатися з іменем файлу (без розширення).
- Метод `main` з сигнатурою `public static void main(String[] args)` є стандартною точкою входу; у JDK 25 допускаються також «безіменні» класи (unnamed classes) для коротких програм.
- Пакет (`package`) — простір імен для організації класів; відповідає структурі директорій.

#### Компіляція та запуск у командному рядку

```bash
# Компіляція
javac HelloWorld.java

# Запуск (вказувати .class НЕ потрібно)
java HelloWorld

# З пакетом
javac -d out src/ua/edu/example/HelloWorld.java
java -cp out ua.edu.example.HelloWorld
```

#### Стандартна бібліотека та модульна система

JDK 25 організований за модульною системою (JPMS, введеною у JDK 9). Модуль `java.base` завантажується автоматично і містить пакети `java.lang`, `java.util`, `java.io` та ін. Клас `java.lang.System` надає поля `in`, `out`, `err` для стандартного I/O.

#### Типи даних та оператор `var`

Для локальних змінних з Java 10 підтримується виведення типу через `var`:

```java
var message = "Привіт!"; // тип String виводиться автоматично
var count = 42;           // тип int
```

#### Безіменні класи (JDK 25)

У JDK 25 стабілізовано можливість запускати програми без явного оголошення класу та методу `main`:

```java
// Файл: Hello.java — запускається командою: java Hello.java
void main() {
    System.out.println("Безіменний клас у JDK 25!");
}
```

Це зручно для навчальних прикладів; у продуктивному коді слід використовувати повну форму.

### Хід виконання роботи

1. Перевірити встановлення JDK: виконати команду `java -version` та `javac -version` у терміналі. Переконатися, що версія відповідає JDK 25.
2. Ознайомитися зі структурою директорії JDK: переглянути вміст `JAVA_HOME/bin`, `JAVA_HOME/lib`. Визначити призначення ключових інструментів (`javac`, `java`, `jar`, `javadoc`).
3. Ознайомитися з навчальним репозиторієм `java-iot`: клонувати або відкрити у браузері файл [`core/src/main/java/ua/crowpi/core/launcher/Launcher.java`](https://github.com/AndriyLutskiv/java-iot/blob/main/core/src/main/java/ua/crowpi/core/launcher/Launcher.java). Проаналізувати структуру класу, точку входу `main`, аргументи командного рядка.
4. Переглянути інтерфейс [`CrowPiProject.java`](https://github.com/AndriyLutskiv/java-iot/blob/main/core/src/main/java/ua/crowpi/core/CrowPiProject.java) — звернути увагу на оголошення інтерфейсу, методів та їхні типи повернення.
5. Налаштувати IDE (IntelliJ IDEA або VSCode з розширенням Extension Pack for Java): створити новий Java-проєкт, вказати JDK 25 як SDK.
6. Створити файл відповідно до свого варіанту індивідуального завдання. Дотримуватися структури: оголошення пакету, оголошення класу, метод `main`.
7. Скомпілювати програму через IDE або командою `javac <ім'я файлу>.java` у терміналі. Проаналізувати повідомлення про помилки, якщо вони виникнуть.
8. Запустити програму та перевірити відповідність виводу умові завдання.
9. Навмисно внести синтаксичну помилку (наприклад, пропустити крапку з комою) та проаналізувати повідомлення компілятора.
10. Виконати програму з аргументами командного рядка (`java <Клас> arg1 arg2`), якщо передбачено варіантом. Переконатися, що `args[0]`, `args[1]` обробляються коректно.
11. Згенерувати документацію Javadoc командою `javadoc -d doc <ім'я файлу>.java`. Переглянути результат у браузері.
12. Оформити звіт: зафіксувати результати компіляції та запуску (знімки екрану або текстовий вивід).

### Індивідуальні завдання

Розробити програму мовою Java відповідно до варіанту. Кожна програма: один клас, точка входу `main`, коректно компілюється та виконується у JDK 25.

**Варіант 1.** Написати програму, що виводить інформацію про студента (прізвище, ім'я, група, рік вступу) у вигляді форматованої таблиці за допомогою `System.out.printf`. Дані задати константами у методі `main`.

**Варіант 2.** Написати програму, що приймає з командного рядка два цілих числа (аргументи `args[0]` та `args[1]`) і виводить їх суму, різницю, добуток та частку. Передбачити перевірку кількості аргументів.

**Варіант 3.** Написати програму-калькулятор температур: приймає значення у градусах Цельсія з командного рядка (`args[0]`), виводить відповідне значення у Фаренгейтах та Кельвінах з поясненнями.

**Варіант 4.** Написати програму, що виводить таблицю множення (10×10) у форматі сітки за допомогою вкладених циклів `for` та `System.out.printf` для вирівнювання стовпців.

**Варіант 5.** Написати програму, що виводить ASCII-арт прямокутника задaних розмірів (рядки та стовпці приймаються з `args[0]` та `args[1]`) із символу `*`. Межі — `*`, внутрішня частина — пробіли.

**Варіант 6.** Написати програму, що обчислює та виводить перші 20 чисел Фібоначчі. Використати цикл `for` та дві змінні для зберігання попередніх значень. Форматувати вивід: кожне число з порядковим номером.

**Варіант 7.** Написати програму, що приймає рядок з `args[0]` і визначає, чи є він паліндромом (читається однаково в обох напрямках). Ігнорувати регістр символів. Вивести результат з поясненням.

**Варіант 8.** Написати програму-лічильник: виводить числа від 1 до 100, але замість чисел, кратних 3, виводить «Fizz», кратних 5 — «Buzz», кратних 15 — «FizzBuzz» (класична задача FizzBuzz).

**Варіант 9.** Написати програму, що обчислює факторіал числа, отриманого з `args[0]`. Передбачити перевірку діапазону (0–20) та повідомлення про помилку при некоректному вводі. Використати тип `long`.

**Варіант 10.** Написати програму, що виводить просте зображення будиночка символами ASCII: дах (`/\`), стіни (`|`), вікно (`[]`), двері (`||`). Розміри фіксовані, оголошені як константи (`static final int`).

**Варіант 11.** Написати програму, що приймає три числа з аргументів командного рядка і знаходить найбільше з них без використання методу `Math.max`. Вивести результат з поясненням порівнянь.

**Варіант 12.** Написати програму, що перевіряє, чи є число, передане у `args[0]`, простим (має ділитися лише на 1 та на себе). Вивести повідомлення та обґрунтування перевірки.

**Варіант 13.** Написати програму-конвертор одиниць: приймає відстань у кілометрах (`args[0]`) і виводить її у милях, морських милях та метрах. Коефіцієнти оголосити як `static final double`.

**Варіант 14.** Написати програму, що виводить діагональний хрест `X` розміром N×N (N приймається з `args[0]`, непарне ціле від 3 до 15). Клітини діагоналей — `*`, решта — пробіли.

**Варіант 15.** Написати програму, що читає ціле число з `args[0]` і виводить всі його дільники, а також визначає, чи є число «досконалим» (сума дільників, крім самого числа, дорівнює числу). Приклад: 6 = 1+2+3.
